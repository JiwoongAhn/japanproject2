# PRD — 일본 대학생을 위한 간편 멀티앱

> **버전:** v1.2
> **작성일:** 2026-04-10
> **앱 이름:** Unipas (ユニパス)

---

## 1. 프로젝트 개요

일본 대학생들이 매일 사용하는 기능(시간표, 과제, 학교 정보)을
하나의 앱으로 통합한 **올인원 캠퍼스 생활 도우미 앱**

> 앱 이름 **Unipas (ユニパス)** — University + Pass. "이 앱 하나면 다 된다"는 의미.
> 이름 변경이 필요하면 이 파일 상단 + app.json의 `name` 항목만 바꾸면 됩니다.

---

## 2. 문제 정의

일본의 대학생들은 학교생활에 필요한 정보와 기능이 여러 앱·사이트에 분산되어 있어
매번 앱을 전환하거나 포털에 따로 로그인해야 하는 불편함을 겪고 있다.

**구체적인 불편함:**
- 시간표 확인을 위해 매번 학교 포털에 로그인해야 함
- 과제 마감일을 따로 메모해두지 않으면 놓치기 쉬움
- 수업 관련 질문이나 학교 정보를 얻을 커뮤니티가 없음
- 유학생·신입생·복학생은 학교 정보를 얻기 더욱 어려움

---

## 3. 목표 및 성공 지표

### 목표
아무런 정보가 없는 **유학생, 신입생, 복학생**이 쉽고 빠르게
학교 정보를 얻고, 시간표를 만들고, 과제를 관리할 수 있게 한다.

### 성공 지표 (국사관대학교 기준)

| 시점 | 목표 |
|------|------|
| 론칭 후 1개월 | 이용자 100명 달성 |
| 론칭 후 3개월 | 이용자 1,000명 달성 |
| 론칭 후 3개월 | 광고 수익 10만 엔 달성 |

## 5. MVP 기능 정의

> **MVP 원칙:** 4단계를 순서대로 완성한다. 커뮤니티·WebView는 Post-MVP.

### 단계별 개발 순서

```
[전제] Supabase 연결 (테이블 구조 준비)
   ↓
[1단계] 시간표  →  [2단계] 과제  →  [3단계] 홈 화면
```

---

### 1단계: Supabase 연결

**목표:** 앱이 Supabase와 연결되고, 데이터를 저장할 테이블 구조가 준비된다.

**Acceptance Criteria:**
- [ ] `schema.sql` 파일에 4개 테이블 DDL 작성 (users, courses, assignments, timetable_slots)
- [ ] `supabase.js`에 환경변수 구조 준비 (API 키는 나중에 입력)
- [ ] 앱 실행 시 Supabase 연결 오류 없이 로딩됨

**이번 단계 제외:**
- API 키 입력 (유저가 직접 Supabase 대시보드에서 복사해서 입력)

---

### 2단계: 시간표 관리

**사용자 스토리:**
> "학생으로서, 나는 앱에서 수업을 추가해 시간표를 만들고, 언제든지 수업을 삭제할 수 있다."

**Acceptance Criteria:**
- [ ] 수업 추가 폼: 과목명, 요일(월~금), 교시(1~8교시), 교수명 입력 가능
- [ ] 저장 후 요일×교시 그리드에 수업 블록이 자동 배치됨
- [ ] 수업 블록 탭 → 삭제 확인 다이얼로그 → 삭제 가능
- [ ] 수업이 없는 교시는 빈 셀로 표시

**이번 단계 제외:**
- 시간 겹침 자동 감지 및 경고 (Post-MVP)
- 수업별 색상 구분 (Post-MVP)

---

### 3단계: 과제 관리

**사용자 스토리:**
> "학생으로서, 나는 수업별 과제를 등록하고, 마감일까지 남은 날을 확인하며, 제출 여부를 기록할 수 있다."

**Acceptance Criteria:**
- [ ] 과제 등록: 제목, 마감일(날짜 선택), 연결 수업 선택 가능
- [ ] 상태 토글: 미제출(🔴) → 제출 완료(🟢) → 기한 초과(⚫) 순환
- [ ] 과제 목록에 D-day 카운트다운 표시 (예: D-3, D-1, D-day, 기한 초과)
- [ ] 기한 초과된 과제는 자동으로 상태 변경

**이번 단계 제외:**
- 푸시 알림 (Post-MVP)

---

### 4단계: 홈 화면

**사용자 스토리:**
> "학생으로서, 앱을 열면 오늘 수업과 마감 임박한 과제를 한눈에 확인할 수 있다."

**Acceptance Criteria:**
- [ ] 오늘 요일 기준으로 오늘 수업 목록 카드 표시
- [ ] D-3 이하 마감 과제 목록 카드 표시
- [ ] 수업/과제 데이터가 없을 때 빈 상태 메시지 표시 (예: "오늘 수업이 없어요")

## 7. 디자인 원칙

### 7-1. 기본 방향
- **모바일 우선:** 모든 핵심 기능은 스마트폰에서 문제없이 동작
- **한 눈에 들어오는 UI:** 정보를 꽉 채우지 않고, 핵심만 크게 표시
- **일본 사용자 특성 반영:** 익명 문화, 깔끔한 디자인 선호

### 7-2. UI 스타일 — 토스(Toss) 스타일
| 원칙 | 적용 방법 |
|------|----------|
| 흰 배경 + 넓은 여백 | 정보를 꽉 채우지 않고 숨 쉬게 |
| 큰 글씨·굵은 폰트 | 한눈에 읽히는 제목 |
| 파란 포인트 컬러 | 버튼·강조 요소에만 사용 |
| 카드 1개 = 정보 1개 | 카드 하나에 여러 정보 넣지 않기 |
| 행동 유도 명확 | 버튼이 뭘 하는지 바로 알 수 있게 |

### 7-3. 컬러 팔레트
| 역할 | 색상 코드 |
|------|----------|
| Primary (버튼·강조) | `#3182F6` |
| Primary Light (배지 배경) | `#EBF3FE` |
| Success (제출 완료) | `#05C072` |
| Warning (마감 임박) | `#FF8A00` |
| Danger (기한 초과) | `#F04438` |
| Background (앱 배경) | `#F2F4F6` |
| Surface (카드 배경) | `#FFFFFF` |
| Text Primary | `#191F28` |
| Text Secondary | `#8B95A1` |

## 8. 핵심 사용자 흐름

```
앱 첫 실행
  → 대학 선택 화면
  → 회원가입 / 로그인
  → 메인 홈 화면

메인 홈 화면
  → 오늘 수업 확인 → 시간표 탭으로 이동
  → 과제 확인 → 과제 탭으로 이동

시간표 탭
  → 수업 추가 → 그리드 자동 배치
  → 수업 삭제

과제 탭
  → 과제 등록 → 마감일 D-day 표시
  → 상태 토글 (미제출 → 완료 → 초과)
```

## 9. 화면 구조

```
App
└── AppNavigator
    ├── AuthStack (로그인 전)
    │   ├── SplashScreen           ← 로딩 화면

    │   ├── LoginScreen            ← 로그인
    │   └── SignUpScreen           ← 회원가입
    │
    └── MainTab (하단 탭 4개)
        ├── 홈        HomeScreen
        ├── 시간표    TimetableScreen / CourseAddScreen
        ├── 과제      AssignmentScreen / AssignmentAddScreen
        └── 게시판    (Post-MVP — 현재 빈 화면 유지)
```

## 11. 비기능 요건

- **익명성:** 익명 게시글은 작성자 정보를 어디에도 표시하지 않음 (Post-MVP)
- **개인정보:** 일본 개인정보보호법(個人情報保護法) 준수
- **시간대:** Asia/Tokyo (JST, UTC+9)
- **날짜 형식:** YYYY年MM月DD日
- **언어:** 일본어 (ja) 기본
