# PRD — 일본 대학생을 위한 간편 멀티앱

> **버전:** v1.1
> **작성일:** 2026-03-28
> **앱 이름:** Unipas (ユニパス)

---

## 1. 프로젝트 개요

일본 대학생들이 매일 사용하는 기능(시간표, 과제, 커뮤니티, 학교 정보)을
하나의 앱으로 통합한 **올인원 캠퍼스 생활 도우미 앱**

> 앱 이름 **Unipas (ユニパス)** — University + Pass. "이 앱 하나면 다 된다"는 의미.
> 이름 변경이 필요하면 이 파일 상단 + app.json의 `name` 항목만 바꾸면 됩니다.

---

## 2. 문제 정의

일본의 대학생들은 학교생활에 필요한 정보와 기능이 여러 앱·사이트에 분산되어 있어
매번 앱을 전환하거나 포털에 따로 로그인해야 하는 불편함을 겪고 있다.

**구체적인 불편함:**
- 시간표 확인을 위해 매번 학교 포털에 로그인해야 함
- 과제 마감일을 따로 메모해두지 않으면 놓치기 쉬움
- 수업 관련 질문이나 학교 정보를 얻을 커뮤니티가 없음
- 유학생·신입생·복학생은 학교 정보를 얻기 더욱 어려움

---

## 3. 목표 및 성공 지표

### 목표
아무런 정보가 없는 **유학생, 신입생, 복학생**이 쉽고 빠르게
학교 정보를 얻고, 시간표를 만들고, 커뮤니티에 참여할 수 있게 한다.

### 성공 지표 (국사관대학교 기준)

| 시점 | 목표 |
|------|------|
| 론칭 후 1개월 | 이용자 100명 달성 |
| 론칭 후 3개월 | 이용자 1,000명 달성 |
| 론칭 후 3개월 | 광고 수익 10만 엔 달성 |

## 4. 핵심 사용자

### 페르소나 A — 학생

| 항목 | 내용 |
|------|------|
| 대상 | 국사관대학교 재학생 (유학생·신입생·복학생 포함) |
| 주요 니즈 | 시간표 관리, 과제 알림, 익명 커뮤니티, 학교 공지 확인 |
| 사용 맥락 | 이동 중 스마트폰으로 빠르게 확인 |

**사용자 스토리:**
> "학생으로서, 나는 나의 시간표를 자유롭게 만들 수 있고,
> 익명의 커뮤니티를 사용할 수 있으며, 학교 정보를 손쉽게 얻을 수 있다."

### 페르소나 B — 교직원·교수

| 항목 | 내용 |
|------|------|
| 대상 | 학교 관계자 |
| 주요 니즈 | 학생들과 동등하게 커뮤니티 참여 |
| 익명성 | 학생과 동일하게 익명성 보장 |

**사용자 스토리:**
> "교직원으로서, 나는 학생과 동등하게 앱을 사용할 수 있으며,
> 누구나 익명성을 보장받는다."

## 5. 핵심 기능 (MVP)

복잡하게 만들지 않는다. 이 4가지가 완벽하게 동작하면 된다.

### 5-1. 홈 화면
- 오늘의 수업 목록 요약
- 마감 임박한 과제 (D-3 이하) 강조 표시
- 최근 커뮤니티 공지 1~2개

### 5-2. 시간표 관리
- 수업 추가 → 요일/교시 그리드에 자동 배치
- 시간 겹침 자동 감지 및 경고
- 수업별 색상 구분

### 5-3. 과제 관리
- 수업별 과제 등록 (제목, 마감일)
- 상태 표시: 미제출 / 제출 완료 / 기한 초과
- D-day 카운트다운 + 마감 1일 전 푸시 알림

### 5-4. 커뮤니티 게시판
- 학교별 독립 게시판
- 익명 / 실명 선택 가능
- 카테고리: 전체 / 질문 / 장터
- 댓글, 좋아요 기능

### 5-5. 학교 정보 (WebView)
- 학교 포털 공지사항을 앱 내에서 열람
- 포털 로그인 정보로 학교 소속 인증

## 7. 디자인 원칙

### 7-1. 기본 방향
- **모바일 우선:** 모든 핵심 기능은 스마트폰에서 문제없이 동작
- **한 눈에 들어오는 UI:** 정보를 꽉 채우지 않고, 핵심만 크게 표시
- **일본 사용자 특성 반영:** 익명 문화, 깔끔한 디자인 선호

### 7-2. UI 스타일 — 토스(Toss) 스타일
| 원칙 | 적용 방법 |
|------|----------|
| 흰 배경 + 넓은 여백 | 정보를 꽉 채우지 않고 숨 쉬게 |
| 큰 글씨·굵은 폰트 | 한눈에 읽히는 제목 |
| 파란 포인트 컬러 | 버튼·강조 요소에만 사용 |
| 카드 1개 = 정보 1개 | 카드 하나에 여러 정보 넣지 않기 |
| 행동 유도 명확 | 버튼이 뭘 하는지 바로 알 수 있게 |

### 7-3. 컬러 팔레트
| 역할 | 색상 코드 |
|------|----------|
| Primary (버튼·강조) | `#3182F6` |
| Primary Light (배지 배경) | `#EBF3FE` |
| Success (제출 완료) | `#05C072` |
| Warning (마감 임박) | `#FF8A00` |
| Danger (기한 초과) | `#F04438` |

| Surface (카드 배경) | `#FFFFFF` |
| Text Primary | `#191F28` |
| Text Secondary | `#8B95A1` |

## 8. 핵심 사용자 흐름

```
앱 첫 실행
  → 대학 선택 화면
  → 학교 포털 로그인 (WebView)
  → 메인 홈 화면

메인 홈 화면
  → 오늘 수업 확인 → 시간표 탭으로 이동
  → 과제 확인 → 과제 탭으로 이동
  → 커뮤니티 글 확인 → 게시판 탭으로 이동

시간표 탭
  → 수업 추가 → 그리드 자동 배치
  → 수업 탭 → 과제 등록 가능

게시판 탭
  → 글 목록 → 글 상세 → 댓글 작성
  → 새 글 작성 (익명/실명 선택)
```

## 9. 화면 구조

```
App
└── AppNavigator
    ├── AuthStack (로그인 전)
    │   ├── SplashScreen          ← 로딩 화면
    │   ├── UniversitySelectScreen ← 대학 선택
    │   └── LoginScreen           ← 포털 로그인 (WebView)
    │
    └── MainTab (하단 탭 4개)
        ├── 홈        HomeScreen
        ├── 시간표    TimetableScreen / CourseAddScreen
        ├── 과제      AssignmentScreen
        └── 게시판    PostListScreen / PostDetailScreen / PostWriteScreen
```

## 11. 비기능 요건

- **익명성:** 익명 게시글은 작성자 정보를 어디에도 표시하지 않음
- **개인정보:** 일본 개인정보보호법(個人情報保護法) 준수
- **시간대:** Asia/Tokyo (JST, UTC+9)
- **날짜 형식:** YYYY年MM月DD日
- **언어:** 일본어 (ja) 기본
