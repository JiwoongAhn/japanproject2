# Google Play 비공개 테스트(Closed Testing) 가이드 — A5

> 신규 **개인** 개발자 계정은 프로덕션 출시 전 **테스터 12명 / 14일 연속** 비공개 테스트가 필수입니다.
> (2024.12 정책 완화로 20명→12명. 2026 현재 유효. 법인 계정은 면제)
> 이 14일이 출시까지 가장 긴 대기 구간이므로 **테스터 모집을 가장 먼저** 시작하세요.

---

## 1. 핵심 규칙 (반드시 충족해야 프로덕션 신청 버튼 활성화)

| 항목 | 요건 |
|---|---|
| 테스터 수 | **최소 12명**이 비공개 테스트에 **옵트인(참여)** 한 상태 유지 |
| 기간 | **연속 14일** (중간에 테스터가 12명 미만으로 떨어지면 카운트 리셋 위험) |
| 활동 | 테스터가 실제로 앱을 설치/실행하는 것이 이상적 (단순 옵트인만으로도 카운트되나, 활동 있으면 안전) |
| 대상 | Google 계정(Gmail) 이메일. 본인 외 **순계정 12명** 필요(본인 계정은 카운트 모호 → 여유 있게 13~14명 권장) |

> ⚠️ **14일은 "옵트인한 테스터 12명이 유지된 날"부터 카운트**됩니다. 12명을 다 모은 그 시점이 D-day 시작이라, **모집을 빨리 끝낼수록 출시가 빨라집니다.**

---

## 2. Play Console 설정 단계 (빌드 완료 후)

1. **Play Console → 테스트 및 출시 → 비공개 테스트** 진입
2. **새 트랙 만들기** (또는 기본 "비공개 테스트" 트랙 사용)
3. **테스터 그룹 만들기**: "테스터" 탭 → 이메일 목록 그룹 생성 → 모집한 12~14명의 Gmail 주소 입력
4. **새 버전 만들기** → A4-2에서 빌드한 **AAB 업로드** (`eas build` 결과물)
5. **출시 노트(릴리스 노트)** 입력 (일본어 — 아래 템플릿 참고)
6. **검토 → 비공개 테스트 트랙으로 출시**
7. **옵트인 URL 복사** → 테스터들에게 배포 (이 링크로 들어가 "테스터 되기" 눌러야 설치 가능)

## 6. 진행 체크리스트

- [ ] 테스터 12명(여유 13~14명) Gmail 주소 수집
- [ ] (빌드 완료 후) Play Console 비공개 테스트 트랙에 AAB 업로드 ← **A4-3**
- [ ] 테스터 그룹에 이메일 등록
- [ ] 릴리스 노트 입력 후 트랙 출시
- [ ] 옵트인 URL 복사 → 테스터들에게 안내문 + 링크 배포
- [ ] 12명 옵트인 확인 → **여기서부터 14일 카운트 시작**
- [ ] 14일 경과 + 12명 유지 → 대시보드 "프로덕션 액세스 신청" 버튼 활성화 확인
- [ ] 프로덕션 신청 → 심사 → 정식 출시
