# Google Play 데이터 보안(Data safety) 답변 (UniOne)

> Play Console → 앱 콘텐츠 → **데이터 보안**. 코드/DB 스키마 기준(app-store-privacy.md와 정합).
> Play는 iOS App Privacy와 질문이 다르므로, 콘솔 설문을 보며 아래 답을 입력.

## 1. 공통 답변 (가장 중요)
| 질문 | 답 | 근거 |
|---|---|---|
| 제3자와 데이터를 **공유**하나요? | **아니요** | 광고·분석 SDK 없음 |
| 전송 중 **암호화**되나요? | **예** | Supabase HTTPS |
| 사용자가 데이터 **삭제 요청** 가능? | **예** | 계정 삭제/문의 경로 제공 |
| 데이터 수집을 위한 **추적**? | **아니요** | 서드파티 추적 없음 |

## 2. 수집하는 데이터 유형 (Play 카테고리 매핑)
| Play 데이터 유형 | 우리 항목 | 수집 목적 | 필수/선택 |
|---|---|---|---|
| 개인정보 > 이메일 주소 | 학교 이메일(`@***.ac.jp`) | 앱 기능·계정 관리 | 필수 |
| 개인정보 > 사용자 ID | 계정 ID(Supabase)·닉네임 | 앱 기능 | 필수 |
| 사진 및 동영상 > 사진 | 게시판 첨부 사진 | 앱 기능 | 선택 |
| 메시지 > 기타 인앱 메시지 | 게시글·댓글 | 앱 기능 | 선택 |
| 기기 또는 기타 ID | Expo 푸시 토큰 | 푸시 알림 | 선택 |

## 3. 콘솔에서 함께 판단할 항목
- 시간표·과제·수업평가·manaba 통지 내용 = **본인 입력 콘텐츠**. Play에 딱 맞는 카테고리 없음 → 필요 시 "기타 사용자 생성 콘텐츠"로. 모두 **본인 계정 저장·공유 안 함**.

## 4. 수집하지 않음 (명시적 No)
위치, 주소록 연락처, 건강·피트니스, 금융정보, 검색/브라우징 기록, 사용 데이터(분석), 진단(크래시/성능), 민감정보. → 서드파티 분석/광고/크래시 SDK 미설치, 위치 권한 미사용.
